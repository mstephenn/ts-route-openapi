# Examples

Each example is a self-contained app whose OpenAPI spec is generated by
`ts-route-openapi` from its route registrations and TypeScript types — the
committed `openapi.yaml` files were produced by `npm run openapi`, never
hand-edited.

| Example | What it shows |
| --- | --- |
| [`express/`](./express) | A runnable Express 5 app using the typed-controller + adapter pattern |
| [`fastify/`](./fastify) | The same app on Fastify 5 |
| [`generic/`](./generic) | The minimal framework-agnostic shape the tool reads (no runtime needed) |

## Running an example

```sh
cd express          # or fastify
npm install
npm run openapi     # regenerate openapi.yaml from the source
npm run dev         # boot the real server on :3000
```

```sh
curl -X POST localhost:3000/orders -H 'content-type: application/json' \
  -d '{"productId":"p9","quantity":3}'
curl localhost:3000/orders/o1
curl 'localhost:3000/orders?limit=5&verbose=true'
```

## The pattern: typed controllers + a thin adapter

`ts-route-openapi` documents **plain typed method signatures** — `getById(id:
string): Order`, not `(req, res)` handlers. Idiomatic Express/Fastify handlers
take framework request/response objects, which carry no route-specific type
information, so there is nothing meaningful to extract from them (the tool
will classify `req` as the request body and produce a useless schema).

Instead, the examples register typed controller methods through a small
`createTypedRouter` facade:

```ts
const router = createTypedRouter(app);
router.get('/orders/:id', OrdersController.getById);
router.post('/orders', OrdersController.create);
```

The facade does two jobs at once:

1. **Runtime**: adapts each call — path params, coerced query params, and the
   JSON body are mapped to the method's parameters using the same
   [parameter-classification convention](../README.md#parameter-classification-convention)
   the doc generator uses.
2. **Documentation**: the `router.get('/orders/:id', OrdersController.getById)`
   call-sites are exactly what `ts-route-openapi` scans, so the wiring and the
   docs can never drift apart.

The adapter is ~40 lines and lives in each example
(`src/typed-router.ts`) — copy it into your project and tweak as needed; it is
deliberately not part of the published package.
